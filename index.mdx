ewelink-api is a JavaScript library that lets you control eWeLink-compatible smart home devices — including the popular Sonoff product line — directly from your Node.js scripts, browser apps, or serverless functions. It wraps the eWeLink cloud API and also supports local LAN control via Zeroconf, so your automations keep working even without an internet connection.

## Get started in three steps

<Steps>
  <Step title="Install the package">
    Add ewelink-api to your project using your preferred package manager.

    ```bash
    npm install ewelink-api
    ```
  </Step>
  <Step title="Authenticate with eWeLink">
    Create a connection using your eWeLink email and password. Set the region that matches your account (default is `us`).

    ```javascript
    const ewelink = require('ewelink-api');

    const connection = new ewelink({
      email: 'you@example.com',
      password: 'your-password',
      region: 'us',
    });
    ```
  </Step>
  <Step title="Control your first device">
    Fetch all devices and toggle one on. Every method returns a Promise, so use `async/await` or `.then()`.

    ```javascript
    const devices = await connection.getDevices();
    const deviceId = devices[0].deviceid;

    const result = await connection.setDevicePowerState(deviceId, 'on');
    console.log(result); // { status: 'ok', state: 'on', channel: 1 }
    ```
  </Step>
</Steps>

<Note>
  The default region is `us`. If your eWeLink account is registered in a different region (e.g., `eu` or `as`), you must pass the correct `region` parameter or you will receive a 400/401 error. Use the `getRegion` method if you are unsure which region your account belongs to.
</Note>
