---
title: "Authenticate: email, token, phone, or LAN mode"
description: "ewelink-api supports three authentication methods — email, phone number, or access token — plus a credential-free LAN mode for Zeroconf."
---

ewelink-api authenticates with the eWeLink cloud by passing credentials to the class constructor. There are three cloud authentication methods (email, phone number, and access token) and a fourth option for local LAN control using Zeroconf, which requires no credentials at all. Choose the method that fits your deployment context.

<Note>
  The default region is `us`. If your account is registered in another region, you must pass the correct `region` value or every API call will return a 400 or 401 error. Use `getRegion()` to look up your region if you are unsure.
</Note>

## Constructor options

<Tabs>
  <Tab title="Email + password">
    The most common way to authenticate. Pass your eWeLink account email, password, and region.

    ```javascript
    const ewelink = require('ewelink-api');

    const connection = new ewelink({
      email: '<your ewelink email>',
      password: '<your ewelink password>',
      region: '<your ewelink region>',
    });
    ```
  </Tab>

  <Tab title="Phone number + password">
    If your eWeLink account is registered with a phone number instead of an email address, use `phoneNumber` in place of `email`.

    ```javascript
    const ewelink = require('ewelink-api');

    const connection = new ewelink({
      phoneNumber: '<your phone number>',
      password: '<your ewelink password>',
      region: '<your ewelink region>',
    });
    ```
  </Tab>

  <Tab title="Access token + API key">
    Use a pre-obtained access token and API key instead of raw credentials. This is useful in serverless environments where you retrieve credentials once and reuse them across multiple requests.

    ```javascript
    const ewelink = require('ewelink-api');

    const connection = new ewelink({
      at: '<valid access token>',
      apiKey: '<valid api key>',
      region: '<your ewelink region>',
    });
    ```

    <Warning>
      The access token and API key are invalidated as soon as you log in again with your email and password. If you re-authenticate with credentials, fetch a fresh token and API key before making further requests with this constructor form.
    </Warning>
  </Tab>

  <Tab title="Zeroconf / LAN mode">
    For local network control without any internet connection, load the device and ARP table cache files generated by Zeroconf and pass them directly to the constructor.

    ```javascript
    const ewelink = require('ewelink-api');
    const Zeroconf = require('ewelink-api/src/classes/Zeroconf');

    const devicesCache = await Zeroconf.loadCachedDevices();
    const arpTable = await Zeroconf.loadArpTable();

    const connection = new ewelink({ devicesCache, arpTable });
    ```

    See the [Zeroconf guide](/guides/zeroconf) for instructions on generating the cache files.
  </Tab>
</Tabs>

## Helper methods

### getRegion()

Returns the region associated with your account. This method only works when the class is initialized with email and password.

```javascript
const region = await connection.getRegion();
console.log(region);
```

Response:

```javascript
{
  email: 'user@email.com',
  region: 'us'
}
```

### getCredentials()

Returns your access token, API key, and region. This is useful in serverless contexts where you want to obtain auth credentials once and pass them to individual requests without re-authenticating on every invocation.

```javascript
const auth = await connection.getCredentials();

console.log('access token: ', auth.at);
console.log('api key: ', auth.user.apikey);
console.log('region: ', auth.region);
```

<Warning>
  The access token and API key returned by `getCredentials()` will be invalidated the next time you log in with your email and password.
</Warning>

## Custom APP_ID and APP_SECRET

The library ships with a default APP ID and APP Secret provided by the Sonoff team. If you need to use a different application pair, pass `APP_ID` and `APP_SECRET` in the constructor alongside your credentials.

```javascript
const connection = new ewelink({
  email: '<your ewelink email>',
  password: '<your ewelink password>',
  APP_ID: 'CUSTOM APP ID',
  APP_SECRET: 'CUSTOM APP SECRET',
});
```
